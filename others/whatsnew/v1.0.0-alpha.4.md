## v1.0.0-alpha.4

### New Features
  - **Integrated Git / Source Control (IDE drawer)** - New Source Control tab in the editor drawer (VS Code–style) with full Git support: view staged/unstaged changes, stage/unstage/discard, commit with amend and GPG sign, fetch/pull/push (including force push), stash, branch management (checkout, create, delete, rename), merge/cherry-pick, submodules, tags, commit history with search, and a side-by-side/inline diff viewer with image preview. File tree rows show a colored Git stripe indicating the change status of each file.
  - **Clang language server** - Added a new server plugin that brings C/C++ language features including completions, diagnostics, go-to-definition, hover, signature help, formatting, and clang-tidy integration.
  - **Markdown preview dark/light mode toggle** - The markdown preview toolbar now includes a toggle button to switch between dark and light themes. The preview opens in the same theme as the app by default and can be changed independently at any time.
  - **Ask AI context menu action** - Selecting text in the editor now shows an "Ask AI" button at the start of the context menu. Tapping it opens a dialog where the user types an instruction (e.g. "Explain this", "Refactor this"). On send, the message is forwarded to the MCP agent, the drawer opens, and the Agent tab is focused automatically.
  - **Fix diagnostic with AI** - The diagnostic tooltip now includes an AI button. Tapping it sends the highlighted error or warning text directly to the Ask AI dialog, allowing the user to get AI-powered explanations or fixes for code diagnostics without manually selecting the problematic code.
  - **Adjustable Symbol bar** - Now you can add/remove/customize a symbol bar
  - **Collapsible App bar** - Now you can collapse the app bar to gain more screen space
  - **Plugin configuration API** - Plugins can now declare themselves as configurable via "configurable": true and "configureClass" in their plugin.json. When set, a Configure button appears on the plugin card in the Plugins screen. The IDE invokes the plugin's configure() method, giving plugin developers full freedom to present any UI they choose via PluginApi.ui.
  - **Multi-instance editor workflow** - Added proper multi-instance editor support: each instance can be switched or closed from the New Instance dialog, instance labels now use the project name, and the dialog includes a `New project` button that opens the template wizard and launches the created project in a new editor instance.
  - **Signing Manager bottom sheet** - Added a new Signing tool in the editor bottom sheets for Android projects. It can create release keystores, scan/select existing keystores, place the keystore under the main Android module, write `keystore.properties`, run module signing reports through the Tooling API/Gradle flow, and safely inject a marked signing config block into `build.gradle` / `build.gradle.kts` on demand.
  - **In-editor media preview tabs** - Clicking previewable files in the file tree now opens them directly inside editor tabs instead of launching external apps. Added support for image formats, SVG preview, Android drawable assets, and built-in video playback controls.
  - **XML & SVG thumbnail previews in file tree** - XML drawable and SVG files now display a live thumbnail preview icon directly in the file tree instead of a generic icon, making it easier to identify assets at a glance.

### Fixes & Improvements
  - **Kotlin indexing progress UI** - Kotlin indexing progress can now be collapsed into a small indicator in the app bar and restored with a tap.
  - **IDE Logs performance** - Improved IDE logs responsiveness during long sessions by switching to incremental updates, adding safe limits, and fully disabling capture/storage when IDE logs are turned off.
  - **Recent Projects instance reuse** - Opening a project from Recent Projects now switches to the already-open editor instance instead of creating a duplicate.
  - **SPLIT web preview layering** - Fixed the bottom sheet header overlaying the web preview in SPLIT mode.
  - **File Tree** - Improved initial loading performance, preserved expanded/collapsed state during refreshes, ensured inline rename stays visible above the keyboard, and blocked renaming the root project node.
  - **Updater 404 fix** - Fixed update checks incorrectly returning 404 even when the URL was correct.
  - **Updater changelog infinite spinner** - Fixed the changelog section getting stuck on an infinite loading spinner when the fetch fails. Failures now show a graceful "unavailable" message instead.
  - **Editor bottom sheet** - Fixed the bottom sheet getting squished when the keyboard is open. It now collapses cleanly and returns to its normal state after the keyboard is dismissed.
  - **File Tree** - Fixed file tree not refreshing properly after file operations such as adding, renaming, or deleting. (https://github.com/AndroidCSIDE/ACSIDE/issues/78)
  - **Editing Area** - Fixed the editing area, completion layout, and signature help window staying dark after switching to Light mode. (https://github.com/AndroidCSIDE/ACSIDE/issues/78)
  - **Kotlin LSP JDK alignment** - Updated the Kotlin language server to the latest version, which now respects the project-level OpenJDK instead of forcing OpenJDK 21. This fixes incorrect diagnostics and completions for projects targeting a different JDK version.
  - **Markdown preview image rendering** - Fixed relative image paths in markdown files not loading in the preview. Images referenced with paths such as `screenshots/image.png` now display correctly.
  - **Search & Replace overlay** - Redesigned the Search/Replace panel layout. The search and replace fields now each have their own row with action buttons below them, giving more horizontal space and making the panel easier to use. The search highlight color has also been changed from yellow to a softer blue tint.
  - **Memory leak fixes:**
    - [x] **(open/close project)** - Fixed six sources of memory leaks that caused memory usage to grow with every project open/close cycle. Background workers, listeners, and object references are now properly cleaned up on every exit path.
    - [x] **(foreground services)** - Fixed the Gradle and Compose Preview background services staying in memory after the editor was closed due to a lingering system power reference.
    - [x] **(plugin API)** - Fixed the plugin editor and process bridge holding references to the destroyed editor activity through static fields that were never cleared on exit.
    - [x] **(foreground service binder)** - Fixed the Gradle and Compose Preview services surviving in memory after the editor was closed because the service release call was being skipped when the activity was destroyed before the connection was fully established.
    - [x] **(editor actions menu)** - Fixed the editor context menu keeping the editor and activity in memory after closing because the listener cleanup relied on an event that was not guaranteed to fire on every exit path.
  - **Out of memory crash fix** - Fixed a crash during builds caused by internal messages accumulating in memory indefinitely instead of being dispatched and discarded.
  - **Stale APK install fix** - Fixed the app sometimes installing the previous build's APK after a successful build instead of the new one.
  - **APK lookup improvement** - The app now uses the exact build output location reported by the Tooling API to find the APK after a build, instead of guessing through a list of common paths. This makes APK detection faster, more reliable, and correct across all project layouts including multimodule projects.
  - **Native build** - Native builds, such as Game Activity projects, can now be compiled directly from external storage without needing to move the project to a directory with execution permissions.
  - **Undo/Reso** - Moved from symbol bar to be an overlay below the line, column indicator 
  - **Plugin API wiring** - Fixed the plugin loader attempting to call `wireEditor` during plugin execution, which requires arguments only available to the editor activity. The loader now correctly skips wiring in editor context since the activity handles it, and only wires home-level context when needed.
  - **Compose Preview multi-module source resolution** - Fixed module detection for previewed files by resolving against synced module directories first instead of falling back to `:app`. Compose Preview now correctly loads sources/classpath for multi-module and multiplatform project templates.
  - **IDE OVERALL IMPROVEMENTS** - Improved the IDE memory management to handle massively large projects
